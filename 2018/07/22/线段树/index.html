线段树

发表于 2018-07-22

强大的数据结构（虽然常数很大）

支持区间的log查询以及修改，能够在一些题目中有效地减小复杂度，达到优化的目的。

$4$倍数组，要和离散化有机地配合。

主要题型：

  1. 区间和、最大值、最小值查询，区间修改（包括值的增减和修改）

  2. 优化$dp$，针对题目的条件特征来决定线段数需要维护的值

  3. $LCIS$、区间最大连续和、$LIS$等经典问题

  4. 第K大问题

  5. 线段树的离线访问和修改

  6. 线段树的综合应用

“C a b c”表示给[a, b]区间中的值全部增加c (-10000 ≤ c ≤ 10000)。

“Q a b” 询问[a, b]区间中所有值的和。

题目描述

按顺序贴海报，求最后还能看到多少张海报

思路

离散化+区间标记，注意在覆盖区间时需要将右端点减一，因为我们的右端点表示的是整个区间。

代码（模板3）

题型2

Special Subsequence（ZOJ3349）

题目描述

给定一个序列S，求最长的子序列满足相邻两个数差值不超过d

思路

离散化+dp+线段树

$dp[i]$表示b编号为i的数为末尾的最长子序列。

那么转移方程就比较显然了，我们枚举一个值，在合法区间内找到最大值，再更新dp。

代码

Minimizing maximizer（POJ1769）

题目描述

给定一些区间，要求选出最少的区间来覆盖整个区间

思路

离散化+dp+线段树，对于每一个区间，我们在其中找到最小的次数来更新此区间，最后再将此区间的右端点更新回来。

题型4

Kth number（HDU2665）

题目描述

求区间第k大值

思路

先敲一个归并树（没毛病。。。），然后将第k大问题转化为二分答案，即对于区间$[l,r]$求大于等于一个数的数有多少个，然后可以在线段树的vector中二分搜索就可以了

复杂度：$O(mlognlognlogn)$

####代码（活在超时边缘的我）

Data Structure?（HDU4217）

题目描述

在一个含有1-n的序列中，每次找到第Ki小的数，并把它删除。

思路

其实呢，感觉这题和第k大数好像没什么关系。。

易知，刚开始所有的数都在，我们将存活的数标记为1，那么我们每次找第k大的数只需要找到在哪个位置的前缀和位k就好了，然后在删掉。

代码

题型5

Turing Tree（HUD3333）

题目描述

给定一个长度为n的序列，给定m个查询，每次查询区间[L,R]范围内不同元素的和。

思路

刚开始想和归并树一样，然后T了。

之后想优化归并树，在vector里用二进制优化，相当于分块处理，然后还是T了。。。（写得不好。。。）

本题正解线段树的离线操作，将所有的区间按右端点排序，然后顺序扫描，显然，当我们处理到了区间的右端点的值于之前的值有相同的时候，我们就可以将之前的值删去，然后再将右端点的值加上，这样操作对后面的区间并没有影响，我们已经按照右端点排序过了。

代码

题型6

Memory Control（HDU2871）

题目描述

有1-N个连续内存单元格。有4种操作

1.Reset,清空所有的内存

2.New x,找到连续x个单元格形成一个内存块，输出最小的下标。

3.Free x, 释放第x个单元给所在内存块的开始和截止位置。

4.Get x,输出第x个内存块从哪个单元格开始的。

思路

看到这种题就不慌，然后写着写着就慌了，线段树需要维护的东西非常多。。。

其实仔细思考，我们只需要用线段树来维护Reset和New和Free操作就可以了，Get可以放在线段树中，但是比较麻烦。

这里采用vector骚操作来维护Get操作（我才不是水过去的！！！虽然大佬说我水过去了）虽然可能会超时，但是还是水过去了。。。
